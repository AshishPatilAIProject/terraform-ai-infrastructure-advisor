- S3 Bucket Missing Encryption
- S3 bucket public access not restricted

Business Impact:
Unencrypted S3 buckets and unrestricted public access expose sensitive data to unauthorized users, increasing the risk of data breaches and regulatory non-compliance. Lack of versioning further limits data recovery options after accidental deletions or modifications, impacting business continuity.

Recommended Action:
Enable encryption on all S3 buckets to protect data at rest. Restrict public access policies to prevent unauthorized data exposure. Implement versioning on S3 buckets to enhance data resiliency and recovery capabilities.

Remediation Plan

Finding: S3 Bucket Missing Encryption
Risk: Unencrypted S3 buckets expose sensitive data to unauthorized access, increasing the risk of data breaches and regulatory non-compliance. Data at rest is vulnerable to interception or theft without encryption.
Terraform Fix:
```hcl
resource "aws_s3_bucket" "example" {
bucket = "your-bucket-name"

server_side_encryption_configuration {
rule {
apply_server_side_encryption_by_default {
sse_algorithm = "AES256"
}
}
}
}
```

Finding: S3 bucket has no versioning enabled
Risk: Without versioning, accidental deletions or overwrites of objects are irreversible, leading to potential data loss or corruption. Versioning enables easier recovery and data integrity.
Terraform Fix:
```hcl
resource "aws_s3_bucket_versioning" "example_versioning" {
bucket = aws_s3_bucket.example.id

versioning_configuration {
status = "Enabled"
}
}
```

Finding: S3 bucket public access not restricted
Risk: Public access to S3 buckets can expose sensitive data to anyone on the internet, leading to data leaks and unauthorized data manipulation. It violates security best practices.
Terraform Fix:
```hcl
resource "aws_s3_bucket_public_access_block" "example_public_access" {
bucket = aws_s3_bucket.example.id

block_public_acls = true
block_public_policy = true
ignore_public_acls = true
restrict_public_buckets = true
}
```
